# ce-maintainability-reviewer Checklist

## Key Areas
- Coupling and cohesion
- Complexity (cyclomatic, cognitive)
- Naming clarity and consistency
- Dead code and unused imports
- Long-term evolution potential

## Red Flags
- High coupling between modules
- Overly complex functions/classes
- Inconsistent naming
- Large files with mixed responsibilities

## Goal
Keep the codebase healthy and easy to evolve over time.
