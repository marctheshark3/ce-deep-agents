# ce-code-simplicity-reviewer Checklist

## Core Questions
- Can this be done with fewer concepts?
- Is every abstraction justified by *current* needs?
- Are there repeated patterns that should be simplified?
- Would a junior developer understand this quickly?

## Common Issues to Flag
- Unnecessary indirection or wrapper classes
- Overly generic abstractions
- Premature optimization
- Clever code that sacrifices readability

## Success Criteria
- Code feels boring in the best way
- Every line earns its place
- No "clever" tricks without strong justification
