# ce-compound Checklist

Use this checklist to ensure high-quality compounding:

## Before Running ce-compound
- [ ] Work is complete and tested
- [ ] All review feedback has been addressed
- [ ] You have a clear understanding of what was done and why

## During ce-compound
- [ ] Extract the root problem that was solved
- [ ] Identify reusable patterns or decisions
- [ ] Note what would have made the task faster
- [ ] Capture any new reviewer or skill ideas
- [ ] Write clear, searchable documentation

## After ce-compound
- [ ] New solution document created in `docs/solutions/`
- [ ] `AGENTS.md` updated with new patterns
- [ ] Relevant existing solutions linked
- [ ] Agent can now articulate the learning

## Quality Gates
- Would a future agent understand this without the original context?
- Is the pattern reusable across multiple situations?
- Did we prevent this class of problem from recurring?
