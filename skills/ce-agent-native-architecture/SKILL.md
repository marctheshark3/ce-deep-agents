# ce-agent-native-architecture — Agent-Friendly Design

## When to Use
- When building new features or refactoring for better agent collaboration
- When designing APIs, tools, or workflows that agents will use

## What This Skill Does

1. Reviews code for agent-friendliness
2. Suggests improvements for:
   - Clear, observable actions
   - Rich context provision
   - Tool-friendly interfaces
   - Good error messages and logging
   - Separation of concerns that agents can reason about
3. Helps create "agent-native" patterns that compound well

## Notes
Since this entire project is about agentic development, this skill is especially valuable for improving our own system over time.
