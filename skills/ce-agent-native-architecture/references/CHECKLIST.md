# ce-agent-native-architecture Checklist

## Key Principles
- Action + context parity (agents can observe and act clearly)
- Clear, tool-friendly interfaces
- Good error messages and logging
- Observable behavior
- Separation of concerns agents can reason about

## Common Improvements
- Add better context to function outputs
- Make state changes explicit and observable
- Reduce hidden dependencies
- Improve error surfacing for agents

## Goal
Design systems that work *with* agents, not against them.
