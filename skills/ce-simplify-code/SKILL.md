# ce-simplify-code — Aggressive Simplification

## When to Use
- After implementing a feature, to clean it up
- When code feels bloated or over-engineered
- As a post-`ce-work` step

## What This Skill Does

1. Runs multiple simplicity-focused reviewers in parallel
2. Identifies opportunities to reduce complexity
3. Applies safe simplifications with test verification
4. Produces a cleaner, more maintainable version of the code

## Notes
This is the "make it boring" skill — highly valuable for long-term health.
